# JOB-010 - Profile, Settings & Final Polish

## Goal
Build the profile/settings screen, connect logout behavior, and apply final consistency checks across the MVP.

## Why
The profile screen gives users a place to view mock account information and exit the app. This task also provides a final polish pass to make sure the main MVP flow feels consistent after the dashboard, transactions, detail, and add transaction screens are complete.

This is the last MVP job, so it should focus on finishing the experience without introducing large refactors.

## Scope
This task should only focus on:
- Building the profile/settings screen UI.
- Showing mock user information from the auth state.
- Adding logout behavior.
- Adding simple settings-style rows if useful.
- Performing small visual consistency fixes across already built screens.
- Updating final MVP progress notes.

This task should not include:
- Real account management.
- Real backend integration.
- Password reset flow.
- Profile image upload.
- Push notification setup.
- Theme switching.
- Advanced settings.
- Large UI redesign.
- Large refactors.
- New package installation.

## Requirements
- Replace the profile placeholder screen with a real profile/settings UI.
- Show mock user information from `useAuth`, such as:
  - name
  - email
- Add a clear logout button.
- Logout should call the existing mock auth logout action.
- After logout, the user should return to the auth flow.
- Use existing base components where appropriate:
  - `Screen`
  - `AppText`
  - `Card`
  - `PrimaryButton`
- Add simple settings rows if they improve the screen, such as:
  - Account
  - Preferences
  - App version placeholder
  - MVP status placeholder
- Settings rows should be visual only unless functionality already exists.
- Keep the screen modern, clean, premium, and readable.
- Use consistent spacing, radius, colors, typography, and card styles.
- Use a careful warning/destructive style for logout if the theme supports it.
- Apply small consistency fixes across existing screens only if needed.
- Small consistency fixes may include:
  - spacing alignment
  - typography consistency
  - card radius consistency
  - color usage consistency
  - screen padding consistency
- Do not rewrite completed screens.
- Do not change core logic from previous jobs unless a small bug fix is required.
- If Reanimated already exists in the project, add subtle profile screen entrance animations.
- Do not add Reanimated if it is missing.
- If Reanimated is unavailable, build the screen without animations and report it as a note.
- Update `progress/progress.md` with a final MVP completion note for JOB-010.

## Constraints
- Do not add new packages.
- Do not implement real profile editing.
- Do not implement image upload.
- Do not implement real settings persistence.
- Do not modify mock data files.
- Do not change routing architecture.
- Do not modify authentication internals unless logout integration requires a small fix.
- Do not perform broad refactors.
- Keep final polish changes small and focused.
- Avoid unnecessary comments.
- Preserve Expo Go compatibility.

## Definition of Done
- Profile placeholder is replaced with a real profile/settings screen.
- Mock user name and email are displayed when available.
- Logout button works with the existing auth system.
- User is redirected to the auth flow after logout.
- Settings-style rows are shown if useful.
- Existing base UI system is used consistently.
- Small visual consistency issues are fixed where needed.
- Reanimated is used only if already available.
- No unrelated files are changed.
- `progress/progress.md` is updated with JOB-010 completion.
- Validation commands are run and results are reported.
