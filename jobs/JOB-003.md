# JOB-003 - Mock Data & Type Definitions

## Goal
Create the core TypeScript data models and mock financial data needed for the personal finance tracker app.

## Why
The app needs a reliable data foundation before building dashboard summaries, transaction lists, transaction detail screens, and add transaction flows.

Typed mock data makes the project easier to develop, test, and review without requiring a backend during the MVP phase.

## Scope
This task should only focus on:
- Creating shared TypeScript types for users, transactions, categories, and transaction summaries.
- Creating local mock data for users, transactions, and categories.
- Creating utility helpers for formatting and basic transaction calculations.
- Keeping mock data simple, readable, and easy to update.

This task should not include:
- Real backend integration.
- AsyncStorage persistence.
- Authentication implementation.
- Dashboard UI implementation.
- Transaction list UI implementation.
- Transaction detail UI implementation.
- Add transaction form implementation.
- Reanimated animations.
- Navigation structure changes.

## Requirements
- Create shared TypeScript types inside `types/`.
- Required type files may include:
  - `types/user.ts`
  - `types/transaction.ts`
  - `types/category.ts`
- Define a `User` type for mock user data.
- Define a `Transaction` type with at least:
  - `id`
  - `title`
  - `amount`
  - `type`
  - `categoryId`
  - `date`
  - `description`
- Transaction type must support both income and expense values.
- Define a `TransactionType` union such as:
  - `income`
  - `expense`
- Define a `Category` type with at least:
  - `id`
  - `name`
  - `type`
  - `color`
  - `icon`
- Create mock data inside `data/`.
- Required mock data may include:
  - `data/mockUsers.ts`
  - `data/mockCategories.ts`
  - `data/mockTransactions.ts`
- Mock transactions should include a realistic mix of income and expenses.
- Mock categories should include common finance categories such as:
  - Food
  - Transport
  - Shopping
  - Bills
  - Entertainment
  - Salary
  - Freelance
- Create utility helpers inside `utils/`.
- Required utility helpers:
  - currency formatting helper
  - date formatting helper
  - total income calculation helper
  - total expense calculation helper
  - balance calculation helper
  - category total calculation helper
  - transaction lookup helper by ID
- Utilities should be pure functions.
- Utilities should be typed.
- Use simple Turkish-friendly or locale-aware formatting where appropriate.
- Update `progress/progress.md` with a short note that JOB-003 was completed.

## Constraints
- Do not add new packages.
- Do not use a real API.
- Do not store sensitive financial data.
- Do not implement persistence in this task.
- Do not modify existing screens.
- Do not modify navigation.
- Do not create UI components.
- Keep mock data small enough to review easily.
- Keep helper functions simple and predictable.
- Use TypeScript.
- Avoid unnecessary comments.
- Preserve Expo Go compatibility.
