# JOB-002 - Theme, Constants & Base UI System

## Goal
Create the app’s basic visual foundation by defining theme constants and reusable base UI components.

## Why
A consistent UI system is needed before building real screens like login, dashboard, transactions, and profile. This task ensures colors, spacing, typography, cards, buttons, inputs, and screen wrappers follow the same design language across the app.

It also prevents every future screen from defining its own inconsistent styles.

## Scope
This task should only focus on:
- Creating theme constants for colors, spacing, radius, typography, and shadows.
- Creating reusable base UI components.
- Applying the base UI components only where needed to verify they work.
- Keeping the existing routing structure from JOB-001 intact.

This task should not include:
- Real dashboard implementation.
- Real transaction list implementation.
- Mock data creation.
- Authentication logic.
- AsyncStorage usage.

## Requirements
- Create a theme constants structure inside `constants/`.
- Define app colors based on the UI guidelines:
  - primary blue or indigo
  - secondary soft green
  - expense red or coral
  - neutral background colors
  - text colors
  - border colors
- Define spacing constants.
- Define radius constants.
- Define typography constants.
- Define shadow constants if needed.
- Create reusable base components inside `components/`.
- Required base components:
  - `Screen`
  - `AppText`
  - `PrimaryButton`
  - `Card`
  - `Input`
- Components must use TypeScript.
- Components must use React Native `StyleSheet`.
- Components should be simple and reusable.
- Components should not include screen-specific business logic.
- Components should support common props such as `style`, `children`, and basic interaction props where appropriate.
- Update placeholder screens only enough to confirm the base components render correctly.
- Do not redesign all screens in this task.
- Update `progress/progress.md` with a short note that JOB-002 was completed.

## Constraints
- Do not add new packages.
- Do not use Tailwind.
- Do not introduce a new UI library.
- Do not implement animations yet.
- Do not implement app data logic yet.
- Do not modify the route architecture from JOB-001.
- Keep components minimal and easy to review.
- Avoid unnecessary comments.
- Keep imports at the top of files.
- Use clear component names.
- Preserve Expo Go compatibility.

## Definition of Done
- Theme constants are created and exported.
- `Screen` component exists and can wrap screen content safely.
- `AppText` component exists and supports basic text variants.
- `PrimaryButton` component exists and supports press handling.
- `Card` component exists and can wrap content.
- `Input` component exists and supports basic text input behavior.
- Placeholder screens still compile and render.
- No unrelated files are changed.
- `progress/progress.md` is updated.
- Validation commands are run and results are reported.
