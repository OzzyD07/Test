# JOB-005 - Auth Screens UI

## Goal
Build the login and register screens and connect them to the mock authentication system created in JOB-004.

## Why
Users need a clear entry point into the app before accessing the dashboard and main tab screens. This task turns the placeholder auth routes into usable screens while keeping authentication mock-only and Expo Go compatible.

## Scope
This task should only focus on:
- Building the login screen UI.
- Building the register screen UI.
- Connecting login and register forms to `useAuth`.
- Showing basic loading and error states.
- Navigating between login and register screens.
- Redirecting authenticated users to the main tab flow if needed.

This task should not include:
- Real backend authentication.
- Real password security logic.
- Dashboard implementation.
- Transaction list implementation.
- Transaction detail UI.
- Add transaction form.
- Profile screen implementation.
- Reanimated animations.
- New package installation.

## Requirements
- Replace the placeholder login screen with a usable login form.
- Replace the placeholder register screen with a usable register form.
- Login form should include:
  - email input
  - password input
  - submit button
  - link to register screen
- Register form should include:
  - name input
  - email input
  - password input
  - submit button
  - link to login screen
- Forms should use the base UI components from JOB-002 where appropriate.
- Forms should call the mock auth actions from `useAuth`.
- Show a loading state while auth action is processing.
- Show a readable error message when auth action fails.
- Keep validation simple and local:
  - required email
  - required password
  - required name on register
- On successful login or register, route the user to the main tab flow.
- If the user is already authenticated, prevent unnecessary access to auth screens where practical.
- Keep screen layout modern, clean, premium, and readable.
- Use consistent spacing, typography, and colors from the theme constants.
- Update `progress/progress.md` with a short note that JOB-005 was completed.

## Constraints
- Do not add new packages.
- Do not modify auth service internals unless a small fix is required for screen integration.
- Do not implement real auth security.
- Do not store or expose real credentials.
- Do not implement animations in this task.
- Do not modify dashboard or transaction screens.
- Do not change the route architecture.
- Keep changes focused on auth UI and auth screen behavior.
- Avoid unnecessary comments.
- Preserve Expo Go compatibility.

## Definition of Done
- Login screen is usable.
- Register screen is usable.
- Login form calls mock login successfully.
- Register form calls mock register successfully.
- Loading state is visible during submit.
- Basic validation errors are visible.
- Authenticated users can reach the main tab flow.
- Users can switch between login and register screens.
- No unrelated files are changed.
- `progress/progress.md` is updated.
- Validation commands are run and results are reported.
