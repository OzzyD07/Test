# JOB-001 - Project Base & Routing Skeleton

## Goal
Set up the initial Expo Router navigation skeleton for the app, including root layout, auth routes, tab routes, and the dynamic transaction detail route.

## Why
This task creates the foundation of the app. A clean routing structure is required before building authentication, dashboard, transaction lists, detail pages, forms, and animations.

The project is designed to test Expo Router’s Stack, Tabs, route groups, and dynamic routes, so the routing architecture must be established first.

## Scope
This task should only focus on:
- Creating the initial Expo Router folder and route structure.
- Creating basic placeholder screens for auth, tabs, and transaction detail.
- Setting up root navigation layout.
- Setting up auth layout.
- Setting up tabs layout.
- Adding a dynamic transaction detail route at `app/transaction/[id].tsx`.

This task should not include:
- Real authentication logic.
- AsyncStorage session handling.
- Dashboard UI implementation.
- Transaction list implementation.
- Transaction detail UI implementation.
- Mock data creation.
- Reanimated animations.
- Form handling.
- Backend integration.

## Requirements
- Create or update the Expo Router root layout at `app/_layout.tsx`.
- Create an initial entry route at `app/index.tsx`.
- Create an auth route group or folder for login and register screens.
- Create placeholder auth screens:
  - `app/auth/login.tsx`
  - `app/auth/register.tsx`
- Create an auth layout if needed:
  - `app/auth/_layout.tsx`
- Create a tabs route group:
  - `app/(tabs)/_layout.tsx`
- Create placeholder tab screens:
  - `app/(tabs)/index.tsx`
  - `app/(tabs)/transactions.tsx`
  - `app/(tabs)/add-transaction.tsx`
  - `app/(tabs)/profile.tsx`
- Create a dynamic transaction detail route:
  - `app/transaction/[id].tsx`
- Placeholder screens should clearly display their screen name.
- Navigation links should be minimal and only used where helpful for testing route access.
- The app should compile without TypeScript errors.
- Update `progress/progress.md` with a short note that JOB-001 was completed.

## Constraints
- Do not add new packages.
- Do not implement real UI polish yet.
- Do not implement authentication logic yet.
- Do not create mock transaction data yet.
- Do not create reusable components yet.
- Do not edit native folders.
- Keep placeholder screens simple and readable.
- Use TypeScript.
- Follow Expo Router conventions.
- Keep the changes limited to routing skeleton files only.

## Definition of Done
- Root layout exists and works.
- Auth screens exist as placeholders.
- Tabs layout exists and works.
- Main tab screens exist as placeholders.
- Dynamic transaction detail route exists as a placeholder.
- The app can navigate between the created route sections during development.
- No unrelated files are changed.
- `progress/progress.md` is updated.
- Validation commands are run and results are reported.
